
适合工程师日常使用的提示词：

- 明确任务目标
- 给足上下文
- 固定输出结构
- 给出可执行建议

### 给老代码整理项目文档
```
请基于该文件代码，输出一份面向团队成员的技术文档。

请包含：
1. 这个模块是做什么的
2. 关键输入输出
3. 核心执行流程
4. 依赖关系
5. 易错点
6. 如何扩展或修改

要求：
- 不要只是把代码翻译成中文
- 重点解释设计意图和边界
- 用 Markdown 输出
- 如果某些行为无法从代码中确认，请明确标注推测
```

### 生成组件
```
根据以下需求生成React组件代码：

【技术栈要求】
- React 18 + TypeScript
- 使用函数组件 + Hooks
- 使用 CSS Modules 进行样式管理

【组件需求】
生成一个 Button 按钮组件，支持以下功能：
1. 支持 primary、secondary、danger 三种样式类型
2. 支持 disabled 禁用状态
3. 支持 loading 加载状态（显示旋转图标）
4. 支持 size 属性：small、medium、large
5. 支持 onClick 点击事件
6. 完整的 TypeScript 类型定义

【代码规范】
- 使用 Props 接口定义类型
- 添加 JSDoc 注释
- 导出默认组件
- 不要使用任何外部 UI 库

请直接输出代码，不要解释。
```
